# KB Fitness

PWA workout tracker for the "8 Weeks to a Healthier You" kettlebell program.

## Commands

```bash
npm run dev        # dev server (http://localhost:5173)
npm run build      # tsc + vite build → dist/
npm run preview    # preview production build
npm run test       # vitest run (single pass)
npm run test:watch # vitest watch
npm run lint       # tsc --noEmit (type-check only, no eslint)
```

## Stack

- **React 18 + TypeScript** via Vite 5
- **HashRouter** — offline-safe routing (no server needed)
- **Dexie v4** — IndexedDB ORM; singleton `db` exported from `src/db/schema.ts`
- **vite-plugin-pwa + Workbox** — service worker, offline caching, installable
- **lucide-react** — all icons
- **CSS custom properties** — dark-only design tokens in `src/index.css`

## Key Conventions

**Routing** — HashRouter with these routes:
- `/` Today, `/program`, `/history`, `/exercises`, `/exercises/:id`
- `/workout` ActiveWorkout (BottomNav hidden)
- `/settings` Settings (BottomNav hidden)

**Database** — weight always stored in kg. Display unit (kg/lb) is a preference in localStorage (`kb_weight_unit`). Week is hard-coded to `1` in `useWorkoutSession` — needs dynamic resolution from `useCurrentProgress` if multi-week tracking is added.

**Stepper** — uses `onDecrement` / `onIncrement` callbacks, not `onChange`. Increment/decrement state locally, pass value down.

**useTimer** — uses `setInterval`, not `requestAnimationFrame`. Required for `vi.useFakeTimers()` to work in tests.

**Design tokens** — all in `src/index.css`. Key vars: `--bg`, `--surface`, `--surface-2`, `--border`, `--primary`, `--accent`, `--text`, `--text-muted`, `--danger`.

**Typography classes** — `t-heading`, `t-body`, `t-label`, `t-label-caps`, `t-title` (defined in `src/index.css`).

## Testing

```bash
npm run test
```

- Vitest + jsdom environment
- `fake-indexeddb/auto` imported at top of DB tests — auto-patches IndexedDB
- Timer tests use `vi.useFakeTimers()` / `vi.useRealTimers()` in beforeEach/afterEach
- DB tests clear tables in `beforeEach` via `db.setLogs.clear()` etc.

## Build Notes

- `base: './'` in vite.config — required for subdirectory / file:// deployment
- PWA manifest auto-generated; icons must exist at `public/icons/icon-192.png` and `public/icons/icon-512.png`
- `npm run lint` runs `tsc --noEmit` — fix all type errors before committing
